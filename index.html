What this dapp does

          
    
            
  1. Snapshot this Minima node (full backup, encrypted with a random password).
    2. 
            
  2. Borrow the webwallet seed: import it into this node via a public MegaMMR sync.
    3. 
            
  3. Sweep all recovered balances back to this node's own existing address.
    4. 
            
  4. Restore this node from the snapshot — your original seed and wallet are back exactly as they were.
    5. 
          

          

            Net effect: the recovered funds end up in this node's normal wallet. The webwallet seed never leaves this device. This node's identity is borrowed for ~5 minutes and then put back.
          

          

            Two restarts required. The flow needs you to restart Minima twice (once after import, once after restore). The dapp remembers where you were and picks up automatically when you come back.
          

          

            This dapp is in READ mode.
            Each action (backup, megammrsync, send) will queue in MiniHub → Pending for your approval. The dapp will pause and auto-resume itself when you approve. WRITE mode runs commands without prompting — both work; pick whichever you prefer.
          

          

            
          

          

            Resuming an in-progress recovery? Or want to abandon and clean up?
            
If you've started a recovery already, this dapp normally picks up where you left off. Click the button below to wipe the saved state if something is stuck.

            

              
            

          

        

      


      
      

        

          
Step 1 of 5 — Snapshot this node

          
Encrypted backup of this node


          

            This dapp does NOT read your seed.
            We'll run backup on this node — the resulting .bak file is encrypted with a random password the dapp generates. Your seed never leaves the secure-vault layer.
          


          

            Before continuing, confirm you already have your own off-device backup of this node's seed (paper, hardware wallet, separate device — anything you control). The dapp won't show you the seed and won't ask for it. If you need an absolute last-resort recovery path beyond the .bak file we're about to make, your existing seed backup is it.
          


          

            This node's vault is password-locked. The .bak we create will be locked too. To restore later, you'll need both the .bak password (shown below) AND your node's vault password (which the dapp doesn't know). Make sure you have the latter recorded somewhere safe.

Step 2 of 5 — Paste the webwallet seed

          
Target webwallet seed

          

            This seed is being treated as the target.
            The recovery sweep will move every coin off it. After today, never type this seed into anything ever again.
          

          
Whatever you originally typed into the webwallet — exactly as you typed it. Webwallet seeds can be 24 BIP-39 words, or any custom string (e.g. a passphrase you made up). Case and spaces matter.

      2.

      3. 
              
    3. Either way, your recovered funds are at the destination Mx you swept to and will be spendable as soon as the wallet is restored.
